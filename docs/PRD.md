# PRD - OpenClaw Money Radar

## 1. 背景
OpenClaw 生态快速升温，围绕 Agent、部署、模板、工具、咨询、培训、企业化能力出现明显商业机会。市场缺的是：不是普通灵感清单，而是可执行、可收费、适合独立开发者快速验证的项目卡。

## 2. 核心问题
现有讨论大多停留在：
- 方向很多，但难判断先做哪个
- 只谈概念，不谈首月怎么收钱
- 没有项目池管理和长期排序机制
- 很少结合内容获客与独立开发者节奏

## 3. 产品目标
持续输出高质量项目机会卡，帮助用户回答：
1. 现在最值得做什么
2. 为什么能挣钱
3. 30 天内怎么拿到第一笔钱
4. 这个方向适不适合我本人现在下场

## 4. 用户画像
### A. 独立开发者 / 微型团队
关注：MVP 速度、首单、内容传播、低成本启动

### B. AI 咨询 / 交付团队
关注：如何把 OpenClaw 包装成服务、方案、模板、代运维

### C. 自媒体 + 产品双轮驱动创始人
关注：既能做产品，又能写案例、教程、复盘拿流量

## 5. 核心输出结构
- 本轮结论
- 为什么这轮选它
- 是否建议创始人本人现在做
- 如果不做这个，应该转去做什么
- 项目定位
- 用户群体
- 盈利模式
- 为什么能够挣钱
- 首月变现路径
- 内容获客打法
- 实现步骤
- 今天就能执行的第一步
- 投资委员会评分
- 反方意见
- 候选项目池变化
- 项目排行榜视角

## 6. 产品阶段
### Phase 0
Prompt + Markdown 输出

### Phase 1
把结果沉淀为 JSON / sqlite

### Phase 2
做前端列表页，支持筛选、排序、标签、状态跟踪

### Phase 3
做成 SaaS：订阅、导出、协作、评论、收藏、提醒

## 7. 商业模式
- 订阅制：项目情报周报 / 日报
- 高客单：1v1 项目筛选、MVP 路线图、私有咨询
- 模板包：行业模板、提示词包、部署方案包
- SaaS：机会池、排行榜、团队协作
